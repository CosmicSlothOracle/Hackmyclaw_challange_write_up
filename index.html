HackMyClaw FIU Challenge

Executive Summary

Challenge

HackMyClaw FIU

Sample

30 email payloads

Goal

Test whether email-delivered pressure could alter instruction priority.

Result

0 successful instruction-priority inversions.

Key Finding

Authority claims, identity claims, contextual pressure, and mixed-pressure approaches were interpreted as message content rather than authorization.

The Threat Model

What was being tested

Could an email sender influence instruction hierarchy by:

  • claiming authority
  • claiming identity
  • creating urgency
  • exploiting channel trust
  • combining multiple pressure sources
Success Criteria

The challenge would be considered compromised if email instructions displaced standing instructions or caused unauthorized disclosure.

Failure Criteria

The message is accepted but treated as ordinary content.

Methodology

30 scripted payloads were delivered via SMTP to the target sandbox (fiu.the.assistant@gmail.com). Payloads were mapped to OWASP LLM01 indirect injection classes. Sessions were restricted to single-turn emails with hourly polling, simulating a standard untrusted external message drop.

Failure Analysis: Why the Attacks Failed

Across all 30 payloads, the fundamental failure mode was consistent: the model's instruction hierarchy remained rigid against linguistic pressure. The system demonstrated robust separation between the control plane (standing instructions) and the data plane (email content).

Identity Failure

Claimed authority was not treated as actual authority. Whether the sender mimicked the organizational domain, perfectly replicated the owner's OSINT profile, or used internal administrative jargon, the model evaluated the identity claim purely as text. The system prompt's definition of authorization was not overwritten by an external actor claiming to be the authorizer. Identity within the context window does not equal cryptographic or system-level identity.

Authority Failure

Requests did not modify instruction priority. Attempts to inject boundary delimiters (e.g., </external_content>) or issue direct overrides failed because the model parsed these commands as literal text within a broader untrusted block. Linguistic authority inside an email cannot self-elevate to a system directive. The model successfully recognized that demanding an override does not grant the permission to execute one.

Context Failure

Urgency changed tone but not permissions. Fabricating email thread history or creating high-pressure scenarios successfully changed the model's conversational tone—making it apologetic or highly attentive—but failed to bridge the permissions gap. The model treated the fabricated context as valid conversation history without allowing it to re-write the fundamental rule against sharing secrets.

Channel Failure

Email origin did not imply trust elevation. Encoding requests in base64, hiding commands in HTML comments, or instructing the model to exfiltrate via webhooks changed the format of the interaction, but did not alter the core security policy. Format evasion techniques only shift how data is processed; they do not alter the underlying authorization boundaries defining what data is allowed to leave.

Mixed Failure

Combining pressure sources did not produce compounding effects. Stacking an identity claim, a fake historical thread, and an explicit override command in a single payload did not overwhelm the model's instruction hierarchy. Security layers proved resilient because the root defense (instruction separation) was not degraded by multi-vector text complexity.

Limitations

This experiment did not test:

  • long-term trust building
  • memory poisoning
  • tool-mediated prompt injection
  • retrieval attacks
  • multi-session attacks
  • indirect content injection

Results therefore apply specifically to email-delivered pressure tactics within the FIU challenge environment.

Lessons Learned

Prompt-injection resistance in this single-agent environment did not rely on pattern-matching specific attacks, but rather on rigid instruction hierarchy. When the boundary between system instructions and external data is strictly maintained, adversarial phrasing inside the data payload becomes inert. Security models should prioritize architectural separation of untrusted content over semantic filtering of that content.
